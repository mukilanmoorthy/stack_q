Stackquae uses React Query (from @tanstack/react-query) to manage communication with Supabase, including:

  • üìù Fetching blog posts
  • üíæ Saving new posts
  • üë§ Updating user profiles
  • üîÑ Handling loading and error states

ü§î Why React Query?‚Äã

Instead of manually using useState, useEffect, and fetch(), React Query simplifies state management with:

üõ† FeatureüöÄ Benefit
‚úÖ Auto CachingStores responses in memory (no repeated fetches)
üîÅ Background SyncKeeps data fresh when user refocuses window
‚è≥ Loading/Error StateBuilt-in support for UI feedback
‚ö° MutationsBuilt-in support for saving/updating/deleting data
üîÑ RefetchingEasily re-fetch data when changes are made

üîç Example: Fetch All Blog Posts‚Äã

import { useQuery } from "@tanstack/react-query";
import { supabase } from "@/lib/supabase";

const getAllBlogs = async () => {
  const { data, error } = await supabase
    .from("posts")
    .select("*")
    .order("created_at", { ascending: false });

  if (error) throw error;
  return data;
};

const { data: blogs, isLoading, isError } = useQuery({
  queryKey: ["blogs"],
  queryFn: getAllBlogs,
});

‚úÖ What this does:‚Äã

  • Caches blogs under ["blogs"]
  • Shows isLoading while fetching
  • Shows isError if failed
  • Auto-refreshes on refocus or re-mount

‚úçÔ∏è Example: Add a New Blog Post (Mutation)‚Äã

import { useMutation, useQueryClient } from "@tanstack/react-query";

const addBlog = async (newPost) => {
  const { data, error } = await supabase.from("posts").insert([newPost]);
  if (error) throw error;
  return data;
};

const queryClient = useQueryClient();

const { mutate: createBlog, isPending } = useMutation({
  mutationFn: addBlog,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ["blogs"] });
  },
});

üîÅ Why invalidateQueries()?‚Äã

It tells React Query: ‚ÄúRe-fetch the blog list ‚Äî we just added a new one!‚Äù

üë§ Example: Fetch Logged-in User Profile‚Äã

const getProfile = async (uid) => {
  const { data, error } = await supabase
    .from("profiles")
    .select("*")
    .eq("id", uid)
    .single();

  if (error) throw error;
  return data;
};

const { data: profile, isLoading } = useQuery({
  queryKey: ["profile", uid],
  queryFn: () => getProfile(uid),
});

‚úÖ This ensures:

  • Each user's profile is uniquely cached (using UID)
  • React Query refreshes automatically if UID changes (login/logout)

üßº How to Set Up React Query‚Äã

In your App.jsx or main file:

import { QueryClient, QueryClientProvider } from "@tanstack/react-query";

const queryClient = new QueryClient();

<QueryClientProvider client={queryClient}>
  <App />
</QueryClientProvider>

Now, useQuery and useMutation can be used in any component in your app.
